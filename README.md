An LLVM module pass that performs **cost-model-guided function inlining** with
call-graph-aware frequency analysis, recursion guards, and dead-function
elimination.

---

## What It Does

`FunctionInliningPass` is an LLVM out-of-tree pass plugin that:

1. **Builds a call graph** of the module using LLVM's `CallGraphAnalysis`.
2. **Scores every callee** with a cost model that weighs instruction count,
   control-flow complexity, and call-site frequency.
3. **Inlines** callees whose score falls below the configurable threshold
   (`--inline-threshold-cost`, default `20`).
4. **Guards** against inlining recursive functions (direct or detected via
   call-graph cycles) and external declarations.
5. **Cleans up** unreachable basic blocks in inlined callers and removes
   dead functions from the module.

| opt      | 14          | included with `llvm-14`              |

> LLVM 16/17 with no changes; for LLVM 18+ the `Optional<WeakTrackingVH>`
> edge API may need updating (see IMPLEMENTATION.md).

## How to Build

```bash
chmod +x build.sh run.sh
./build.sh
```

On success you'll see:

```
[OK] Build successful: build/libInlinePass.so
```

---

## How to Run

### Single test case

```bash
./run.sh testcases/test01_basic_leaf.c
```

### All 10 test cases at once

```bash
./run.sh
```

### Custom source file

```bash
./run.sh path/to/your_program.c
```

### Adjust the inlining threshold

The threshold is a compile-time `cl::opt`. Pass it via `opt`'s `-mllvm` flag
or modify `run.sh` to add `--inline-threshold-cost=<N>`:

```bash
opt --load-pass-plugin=build/libInlinePass.so \
    --passes="inline-pass"                    \
    --inline-threshold-cost=10                \
    input.ll -S -o output.ll
```

## Output

For each run the script writes to `results/`:

| File                         | Contents                                   |
|------------------------------|--------------------------------------------|
| `<basename>.ll`              | Original LLVM IR (unoptimised, `-O0`)      |
| `<basename>_opt.ll`          | LLVM IR after InlinePass                   |
| `<basename>.log`             | Per-function INLINE / SKIP decisions + summary |

A side-by-side diff and instruction-count reduction metric are printed to
stdout.

## Quick Expected Results (summary)

| Test | Scenario                     | Expected outcome              |
|------|------------------------------|-------------------------------|
| 01   | Basic leaf function          | Inlined + dead fn removed     |
| 02   | Nested call chain            | Both levels inlined           |
| 03   | High-frequency callee        | Inlined (negative net cost)   |
| 04   | Large function (>threshold)  | Skipped — cost too high       |
| 05   | Direct recursion             | Skipped — recursion guard     |
| 06   | Mutual/indirect recursion    | Skipped — recursion guard     |
| 07   | External library calls       | Skipped — no body available   |
| 08   | Multiple callers             | Leaf inlined at each site     |
| 09   | Unreferenced dead code       | Dead fn removed without inlining |
| 10   | Mixed pipeline               | Selective inline + DCE        |

See `EVALUATION.md` for detailed metrics.
